SECTIONS are made of PARTS. There are several different types of PART, some very specific, others less so. All PARTS share a few common features.

  • All parts have a name and are always a child element of a section.
  • A part may be marked as muted. A muted part does not contribute
    when it's parent section is rendered.
  • Parts have a 'period' which is by default is the same as the
    section duration. The period may be some fixed
    divisor of the section duration. When the section is rendered, the
    part is repeated every period seconds as needed to
    to fill the section's duration. An 8-bar section may have parts
    with 1, 2, 4, or 8 bar periods.

Specific types of PART are described starting on the next page. The following functions apply to all parts.

(PART-P obj)
Predicate, returns true if obj is an instance of PART or one of it's sub-classes.

(MUTE part flag)
Sets the mute status of part to flag. A muted part is not used when it's parent section is rendered. See also Groups.

(UNMUTE part)

(SOLO part)
Set part as unmuted. Mute all other parts in the same section.

(RENDER-ONCE part &key (offset 0))
Generates MIDI events from part specification. Returns nested list of events with form:

            
            ((time1 . event1)
             (time2 . event2)
              ..............
             (timen . eventn))

If the part is muted returns nil. The offset value is a time shift applied to all event times.

(RENDER part &key (repeat 1)(offset 0))
Call RENDER-ONCE repeat times. Returns list of events.

(CLONE part &key (newname "~A-clone")(parent nil)(hook #'identity))
Creates clone of part. Arguments have the same usage as Section clone.

(TRANSPOSE! part x &key (range '(0 127))
Transpose part by x half-steps. If the part is 'non-transposable' TRANSPOSE! will have no effect. The optional range argument sets the possible range of resulting keynumbers. Any keynumber which is transposed outside of range it is transposed by octaves as needed to be in range. Returns part.

(INVERT! part pivot &key (range '(0 127))
Apply key inversion around pivot-point. If the part is 'non-transposable' INVERT!! will have no effect. The optional range argument sets the possible range of resulting keynumbers. Any keynumber which is outside of range it is transposed by octaves as needed to be in range. Returns part.
