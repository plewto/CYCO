CYCO Metric Values

Metric values are used for event duration, and may be expressed directly as non-negative numbers or symbolically.

            
            Basic Metric Values:
              
              W - 4    Whole note
              H - 2    Half note
              Q - 1    Quarter note
              E - 1/2  Eighth note
              S - 1/4  Sixteenth note
              T - 1/8  Thirtysecond note 
              X - 1/16 Sixtyfourth note

The basic values may be modified by appending one of the following to symbols above.

            
               .   - single dot  3/2 original value
               ..  - double dot  7/4 original value
               T   - triplet     2/3 original value
               5   - tuplet      4/5 original value
               +   - increment   Adds 1 tick to original 
               -   - decrement   Subtracts 1 tick from original

Thus 'QT' is a quarter note triplet, while 'E..' is a double dotted eighth note.

The symbol R is used for a rest of any length and has a value of -1. All negative values are "promoted" to -1.

A limited type of metric expression may be used.

            
            [n*]a[+b[+c[+...]]]
            
            Brackets [] indicate optional elements.
            The elements a, b, c are metric values as above,  and the expression a+b+c returns the 
            their sum.  
            
            H+Q+ST = half note+quarter note + sixteenth triplet, or equivalently H.+ST.
            
            The optional term [n*] is a scaler which multiplies the sum by n.
            The scaler term must precede all other terms and is applied after the sum. 
            
            n*(a+b+c).
            
            3*W+Q is 3 times the sum of a whole note and quarter note.
            
            Metric expressions are represented by Lisp symbols and may not include
            spaces:
            
            H+Q+S     - correct
            H + Q + S - incorrect

Metric related functions

            
            (METRIC-P obj)
            
            Predicate, returns true if obj is a valid metric value. 
            Does not recognize metric expressions.
                
                (METRIC-P 13)   -> T
                (METRIC-P 'W.)  -> T
                (METRIC-P 'W+Q) -> NIL
            
            (METRIC obj)
            
            Returns metric value of argument.
            If argument can not be converted, it is used as the return value.
            METRIC is defined for floats, symbols and list.
            
               (METRIC -99)   ->  0  (hum this contradicts statement above?)
               (METRIC 'R)    -> -1
               (METRIC 13)    -> 13
               (METRIC 'Q)    ->  1
               (METRIC 'APE)  -> APE
               (METRIC 'Q+S)  -> Q+S
               (METRIC '(1 2 3 Q. APE))  -> (1 2 3 3/2 APE)
