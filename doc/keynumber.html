CYCO Keynumbers

Previous( Controlball )     Next( Metric Values )     Home     Contents     

Ultimately CYCO keynumbers are MIDI keynumbers, that is integers between 0 and 127 inclusive. All negative keynumbers are "promoted" to -1 and treated as rest.

Keynumbers may also be expressed symbolically with the general form:

            
            RMO
            
            Where R is the root key (white key) and is one of C D E F G A or B.
                  M is optional modifier and may be one of.
                    F  - flat
                    S  - sharp
                       - natural
                  O is octave number between 0 and 10.
            
            C4  - Note C in 4th octave
            CS4 - Note C sharp also in 4th octave.
            
            Enharmonic values are defined for B-sharp, C-flat, E-sharp and F-flat
            
            CF4 = B3 
            BS3 = C4
            
            CYCO octave numbers are 1 octave lower then typical notation.
            C5 is middle C with keynumber 60.   This is to avoid negative octave 
            numbers.
            
            The special keynumber R corresponds to -1 and is treated as a rest.

Functions related to keynumbers:

            
            (KEYNUMBER-P obj)
                Predicate true if obj is a valid key-number.
            
            (KEYNUMBER obj)
               Returns MIDI keynumber between 0 and 127 or -1.
               keynumber is defined for
                 integers
                 symbols
                 list - if argument is a list, return a list.
            
            (REST-P obj)
               Predicate returns true if obj is a keynumber and it is a rest.
            
            (KEYNAME obj)
               Returns symbolic keynumber name for obj.
               In the case where obj may have two names, such as a-flat and g-sharp,
               the sharp value is always returned.
            
               (keyname 34)  ->  AS2
               (keyname 'c4) ->  C4
               (keyname '(34 35 36))  ->  (AS2 B2 C3)
            
            (PITCH-CLASS obj)
               Returns pitch class of keynumber
            
               (pitch-class 60)  -> 0
               (pitch-class 61)  -> 1
               (pitch-class 'g7) -> 7
               (pitch-class '(60 61 62)) -> (0 1 2)
            
            (OCTAVE obj)
               Returns octave of keynumber
            
               (octave 60)  -> 5
               (octave 'g7) -> 7
               (octave '(c4 c5 c6)) -> (4 5 6)
            
            (TRANSPOSE kn x &key (range '(0 127)))
               Transpose kn by x half-steps
               If result is outside of range, transpose by octaves as needed
               to be in range.
            
               Transpose is defined for integer, symbol and list.
            
             (INVERT kn pivot (range '(0 127)))
                Return key inversion of keynumber.
                If result is out of range, transpose by octaves as needed 
                to be in range.
